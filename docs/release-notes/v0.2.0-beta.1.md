# Gitmoot v0.2.0-beta.1

Gitmoot v0.2.0-beta.1 promotes the recent worktree, permission, Claude
readiness, and forkable temp-worker changes into the public beta line.

## Highlights

- Implementation jobs now run in task-owned worktrees instead of mutating the
  registered checkout directly.
- Read-only agents are blocked before implementation starts, with clear
  user-facing guidance instead of write failures.
- Claude Code readiness checks now classify missing OAuth/auth setup and tell
  users to run `claude setup-token` when needed.
- Busy Codex and Claude runtime sessions can fork bounded temporary workers for
  parallel implementation tasks, then merge successful work back.
- Task list/show output now exposes task worktree location and lifecycle state.

## Features

- Task worktree allocation for `gitmoot task run` implementation jobs.
- Checkout mutation serialization for branch/worktree setup.
- Daemon execution from stored task worktrees for implementation and review
  jobs.
- Post-delivery workflow advancement for task implementation jobs.
- Autonomy policy validation for registered agents and managed agent types.
- Runtime adapter policy propagation so Codex/Claude receive the intended
  write/read behavior.
- Permission failure classification for read-only and approval-blocked jobs.
- Claude auth readiness reporting in `gitmoot agent doctor` and plugin checks.
- Parallel session policy configuration with bounded temp-worker forks.
- Temp-worker delegation, merge-back jobs, resource-lock cleanup, and tests.

## Verification

- `GOTOOLCHAIN=go1.26.0 go test ./...`
- Local smoke validation for automatic task worktree routing.
- Real GitHub PR smoke validation for comments, PR state refresh, merge gate,
  and worktree cleanup.
- Herdr-visible parallel worker smoke with same-agent Codex jobs where one job
  used the original runtime session and another used a temporary worker.
- Claude Code OAuth setup was validated with real `claude -p` background/ask
  execution after token setup.

## Known Beta Limits

- Local coordinator delegation now has first-class `gitmoot agent run`,
  `gitmoot agent review`, and `gitmoot agent implement` command paths. They are
  intended to keep worktree, branch, commit, push, PR, and advancement lifecycle
  inside Gitmoot rather than child-agent prompts.
- SkillOpt live pairwise candidate-vs-current promotion validation is still a
  future optional mode.
- Preset prompt delivery for resumed agents can still be optimized further.
- Release assets currently include `gitmoot_linux_amd64` only.
